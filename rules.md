# Régles d'écriture de code

## Règles générales

- Inclure systématiquement le texte de la licence en début de fichier.
- Déduire des exemples d'utilisation, et mettre ces exmples de code en commentaire à la fin du fichier.
- Chaque méthode doit être commentée.

## Componenets

- Un composant (Component) doit toujours avoir une classe qui porte de le nom du fichier.
